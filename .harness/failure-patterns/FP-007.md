# FP-007 用户明确要 Claude 调研时, 不要先本地铺垫

## 现象
用户已经明确指定“让 Claude Code 去看/调研/对比”，但还是先在本地读了一堆文件、搜了一堆上下文，导致当前会话被无关铺垫塞满，真正委托给 Claude 反而变慢。

## 为什么错
这类任务的目的就是把脏活和上下文负担交给 Claude Code。
如果还先本地做大规模铺垫：
- 会污染自己的上下文
- 会浪费用户时间
- 还会削弱 Claude Code 的价值

## 正确做法
1. 用户明确点名 Claude Code 时，直接委托。
2. 只给 Claude 最小必要背景：目标、仓库路径、想回答的问题。
3. 只有在 Claude 返回后，才在本地补少量验证或执行落地动作。
4. 如果需要并行，就把 Claude 任务放后台异步执行，不要先本地铺满上下文。

## 一句话规则
用户明确要 Claude 调研时，先委托 Claude，不要先本地铺垫污染上下文。
