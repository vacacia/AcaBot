# FP-002 异步 E2E 等错完成信号

## 现象
我在 scheduler E2E 里最开始用 `len(gateway.sent) >= 2` 作为等待条件，就继续往下断言：
- scheduler 已触发
- synthetic event 已注入
- agent 已被唤醒
- cancel 已成功

但 `gateway.sent >= 2` 只能说明“消息数量到了”，不等于“scheduler 这条语义链已经完成”。

## 为什么错
异步链路不能等表面现象，必须等语义完成信号。

这里真正要等的是：
- 第二轮 wake run 已经发生
- agent 的 wake handler 已执行

否则测试会：
- 偶尔误过
- 偶尔误挂
- 在机器快慢不同的时候变得不稳定

## 代价
- 测试有竞态
- 通过结果不可信
- 断言失败点会误导排查方向

## 以后怎么避免
1. 先定义“这条链路真正完成”的业务信号。
2. 等待条件必须绑定语义状态，而不是只看计数。
3. 对 scheduler / queue / callback / event 这种异步链路，优先等待：
   - 回调计数
   - 状态位
   - 事件标记
   - 持久化记录
   而不是只等消息数。

## 一句话规则
**异步 E2E 要等业务完成信号，不要等表面计数。**
