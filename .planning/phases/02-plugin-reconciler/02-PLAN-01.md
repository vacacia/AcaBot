### T1.5 — 创建 `plugin_runtime_host.py` [PLUG-06, PLUG-07]

**文件:** `src/acabot/runtime/plugin_runtime_host.py` (新建)

实现:
- `PluginLoadSnapshot` — `@dataclass(frozen=True)`, 字段: `tool_names: list[str]`, `hook_descriptors: list[str]`
- `RuntimeHookRegistry` — 从 `plugin_manager.py` L104-154 移植. `register()` 追加并按 priority 排序, `get()` 过滤 enabled=False
- `PluginRuntimeHost`:
  - 构造函数: `tool_broker: ToolBroker`, `model_target_catalog: MutableModelTargetCatalog | None = None`
  - 内部状态: `_loaded: dict[str, RuntimePlugin]`, `_hook_registry: RuntimeHookRegistry`, `_plugin_hooks: dict[str, list]`, `_plugin_tool_sources: dict[str, str]`
  - `async load_plugin(package: PluginPackage, context: RuntimePluginContext) -> PluginLoadSnapshot`:
    1. 按 `package.entrypoint` 导入并实例化 `RuntimePlugin`
    2. 校验 `plugin.name == package.plugin_id`, 不一致则 raise
    3. 调 `plugin.setup(context)`
    4. 注册 model target (如有), 失败则 rollback: unregister + revalidate + teardown
    5. 注册 hooks: `plugin.hooks()` → `_hook_registry.register()`
    6. 注册 runtime_tools: source = `f"plugin:{package.plugin_id}"`
    7. 注册 legacy tools: `plugin.tools()`
    8. 加入 `_loaded`, 返回 `PluginLoadSnapshot`
  - `async unload_plugin(plugin_id) -> None`:
    1. `tool_broker.unregister_source(f"plugin:{plugin_id}")`
    2. `unregister_plugin_targets(plugin_id)`
    3. `plugin.teardown()`
    4. 精确清理该插件的 hooks, 重建 registry
    5. 从 `_loaded` 移除
  - `async teardown_all() -> None` — 逆序遍历 `_loaded`, 对每个执行 unload 逻辑
  - `loaded_plugin_ids() -> set[str]`
  - `async run_hooks(point, ctx) -> RuntimeHookResult`:
    - 按 priority 排序执行, 单个 hook 异常记日志继续 [PLUG-07]
    - `skip_agent` 短路返回
  - Model target revalidation 逻辑从 `plugin_manager.py` L842-851 + L578-597 完整移植, 保留 rollback 流程

### T1.6 — 创建 `plugin_reconciler.py` [PLUG-05, PLUG-07]

**文件:** `src/acabot/runtime/plugin_reconciler.py` (新建)

实现:
- `PluginReconciler`:
  - 构造函数: `catalog: PackageCatalog`, `spec_store: SpecStore`, `status_store: StatusStore`, `host: PluginRuntimeHost`, `context_factory: Callable[[str, dict[str, Any]], RuntimePluginContext]`
  - `async reconcile_all() -> list[PluginStatus]`:
    1. `catalog.scan()` 取 packages + package_errors
    2. `spec_store.load_all()` 取 specs + spec_errors
    3. 对解析失败的 plugin_id: 若 host 中已加载则先 unload, 生成 `failed` status 并 save
    4. 收集 `all_ids = (packages | specs | host.loaded_plugin_ids()) - error_ids`
    5. 按 `plugin_id` 字母序遍历, 对每个调 `_reconcile()`
    6. 返回所有 status 列表
  - `async reconcile_one(plugin_id) -> PluginStatus`:
    - 单独取该 plugin 的 package + spec, 调 `_reconcile()`
  - `async _reconcile(plugin_id, package, spec) -> PluginStatus`:
    - Spec 存在但 Package 不见 → unload (如已加载) → `uninstalled`
    - 没有 Spec 或 disabled → unload (如已加载) → `disabled`
    - Package + Spec enabled → unload 旧 (如已加载) → merge config → `context_factory()` → `host.load_plugin()` → `loaded` 或异常 → `failed` [PLUG-07]
    - 每个分支都 `status_store.save(status)` 后返回
