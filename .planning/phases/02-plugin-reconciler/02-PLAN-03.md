# Wave 3: WebUI Plugin Management Page

**Phase:** 02 — Plugin Reconciler
**Wave:** 3 of 3
**Requirements:** PLUG-11

## Goal

重写 `PluginsView.vue` 和 `api.ts` 中的插件相关接口, 实现完整的插件管理页面: 列表 + 状态徽章 + enable/disable 开关 + schema 驱动配置表单 + 重新扫描按钮.

## Prerequisites

- Wave 2 全部完成: 新 API 端点已上线并可通过 `GET /api/system/plugins` 等访问
- bot 可正常启动并响应 API 请求

## Tasks

### T3.1 — 更新 `api.ts` 插件接口 [PLUG-11]

**文件:** `webui/src/lib/api.ts`

1. **删除旧接口/类型**:
   - 旧的插件配置相关类型 (如有)
   - 旧的 `/api/plugins`, `/api/system/plugins/config` 相关调用函数

2. **新增类型定义**:
   ```typescript
   interface PluginPackageInfo {
     display_name: string
     version: string
     entrypoint: string
     default_config: Record<string, unknown>
     config_schema: Record<string, unknown> | null
   }

   interface PluginSpecInfo {
     enabled: boolean
     config: Record<string, unknown>
   }

   interface PluginStatusInfo {
     phase: 'disabled' | 'loaded' | 'failed' | 'uninstalled'
     load_error: string
     registered_tools: string[]
     registered_hooks: string[]
     updated_at: string
   }

   interface PluginView {
     plugin_id: string
     package: PluginPackageInfo | null
     spec: PluginSpecInfo | null
     status: PluginStatusInfo | null
     effective_config: Record<string, unknown>
   }
   ```

3. **新增 API 函数**:
   - `fetchPlugins()` → `apiGet<{plugins: PluginView[]}>("/api/system/plugins")`
   - `fetchPlugin(pluginId)` → `apiGet<PluginView>("/api/system/plugins/" + pluginId)`
   - `updatePluginSpec(pluginId, enabled, config)` → `apiPut<PluginView>("/api/system/plugins/" + pluginId + "/spec", {enabled, config})`
   - `deletePluginSpec(pluginId)` → `apiDelete<PluginView>("/api/system/plugins/" + pluginId + "/spec")`
   - `reconcilePlugins()` → `apiPost<{plugins: PluginView[]}>("/api/system/plugins/reconcile", {})`

4. **更新缓存失效映射**:
   - 旧映射 (`/api/system/plugins/config`, `/api/plugins/reload`) → 删除
   - 新映射: 对 `/api/system/plugins/` 开头的 PUT/POST/DELETE 操作, 失效 `["/api/system/plugins", "/api/status"]`

### T3.2 — 重写 `PluginsView.vue` [PLUG-11]

**文件:** `webui/src/views/PluginsView.vue` (完全重写, 原 105 行)

**页面结构:**

1. **页面头部** (ds-page + ds-hero 模式):
   - 标题: "插件管理"
   - 副标题: 简短说明
   - "重新扫描" 按钮 (ds-secondary-button), 调 `reconcilePlugins()`, 带 loading 状态

2. **插件列表** (ds-list 模式):
   - 每个插件一行:
     - `display_name` (package 为 null 时 fallback 到 plugin_id)
     - Phase 徽章: `loaded` (绿色), `disabled` (灰色), `failed` (红色), `uninstalled` (黄色)
     - Enable/disable 开关 (toggle switch), 绑定 `spec?.enabled ?? false`
     - 点击行展开详情面板

3. **展开面板** (每个插件的详情):
   - **来源信息**: plugin_id, version, entrypoint
   - **状态信息**:
     - `registered_tools` 列表 (显示为 tag/chip)
     - `registered_hooks` 列表 (显示为 tag/chip)
     - 如 phase === 'failed': "查看错误" 按钮, 点击弹出 modal 显示 `load_error` 全文 [D-04]
   - **配置编辑区**:
     - 有 `config_schema` 时: schema 驱动表单 (见 T3.3)
     - 无 `config_schema` 时: `effective_config` 的只读 JSON 视图
   - **操作按钮**:
     - "保存配置" (ds-primary-button), 调 `updatePluginSpec()`
     - "恢复默认" (ds-secondary-button), 调 `updatePluginSpec(id, enabled, {})`, 保持 enabled 不变
     - "移除并禁用" (ds-danger-button), 确认对话框后调 `deletePluginSpec()`

4. **错误 Modal** [D-04]:
   - 简单 overlay + 白色面板
   - 显示 `load_error` 全文 (pre 标签, 可滚动)
   - 关闭按钮

5. **Toast 通知** [D-03]:
   - 成功/失败操作后显示 toast (右上角, 3 秒自动消失)
   - 使用简单的 `ref<{message, type}>` + CSS 动画实现, 不引入 toast 库

### T3.3 — Schema 驱动配置表单 [PLUG-11]

在 `PluginsView.vue` 内实现 (或提取为 `webui/src/components/SchemaForm.vue` 组件):

根据 `package.config_schema` 的 JSON Schema 定义, 自动生成表单字段:

| Schema type | 渲染 |
|-------------|------|
| `string` | `<input type="text">` |
| `number` | `<input type="number">` |
| `boolean` | `<input type="checkbox">` (开关样式) |
| `array` of `string` | 可编辑的标签列表 (输入框 + 添加按钮 + 标签显示 + 删除) |
| `object` (嵌套) | 整块 JSON textarea 编辑 |

表单编辑的是 `spec.config` (override 部分):
- 每个字段的 placeholder 来自 `package.default_config` 对应 key
- 用户清空某字段 → 从 `spec.config` 中删除该 key (回退到 default)
- 字段的 `description` 显示为 label 或 tooltip

### T3.4 — 交互流程实现 [PLUG-11, D-03, D-05]

**页面加载:**
- `onMounted()` 调 `fetchPlugins()` 加载列表
- 支持 `peekCachedGet()` 做缓存优先的初始渲染

**Toggle enabled:**
- 切换开关 → 构造新 spec (`{enabled: !current, config: current_config}`) → `updatePluginSpec()`
- 开关立即变为 loading 状态 (disabled + spinner) [D-03]
- 返回后用响应数据更新本地状态

**保存配置:**
- 点击保存 → `updatePluginSpec(id, enabled, edited_config)` → 用返回值更新本地
- 保存按钮 loading 状态

**重新扫描** [D-05]:
- 点击 "重新扫描" → 按钮 loading
- 列表中所有插件临时显示 "reconciling" 过渡状态
- `reconcilePlugins()` 返回后, 用完整列表替换本地状态
- 清除 loading + 过渡状态

**删除 spec:**
- 确认对话框 → `deletePluginSpec()` → 更新本地状态 (插件变为 disabled, config 清空)
